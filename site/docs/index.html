ssbuild

my personal static site builder

Overview

ssbuild is a very opinionated (and hacky) static site builder. It's designed to make it quick and easy for me to create new mini-sites. It'll be replaced by Neopoligen once that app is ready. It lets me get ideas outta my head with less friction in the mean time.

It also give me another rep in designing and building a static site generator.

I wouldn't recommend trying to use it. It's super customized for the way I'm currently working. Feel free to poke around the source code if you're into that kinda thing.

Features

- [x] Runs in the current directory

- [x] Loads every `.html` file in `./content` 
    as a template.

- [x] Skips `.html` files in directories
    that start with a `_` at the root of
    the project. Everything else gets processed
    and its output placed in `./docs` with
    a matching file name/path.

- [x] Empties the `./docs` output directory 
    before each run.

- [x] Runs a local dev server with hot-reload
    when files change.

- [x] Automatically finds an available port
    for the server so multiple instances
    can be run at the same time.

- [] Automatically make `content` and
    `docs` directories if they don't exist.

- [] Automatically make an `index.html` file
    when making `content` directory if it 
    doesn't exist. 



- [x] Pulls .html files from `./content`,
processes them as templates in MiniJinja
and drops the output in `./docs` with the same
file name/path that the source file had in `./content`.

- [] Copies non .html files from `./content/`
directly to `./docs` with the same file name/path.
